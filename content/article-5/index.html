# The attention in transformer （面经问题总结）

在NLP算法相关的面试里，Transformer和其中的Attention机制显然是重中之重。而相关的高频提问笔者总结主要有如下几个问题形式。

# 1. 问：自注意力(self-attention)的计算公式是怎样的？

答：

Attention (Q,K,V)=softmax⁡(QKTdk)V(1)\text{Attention }(Q, K, V)=\operatorname{softmax}\left(\frac{Q K^{T}}{\sqrt{d_{k}}}\right)V \tag{1} Attention (Q,K,V)=softmax(dk​​QKT​)V(1)

其中Q,K,VQ,K,VQ,K,V为query,key,valuequery,key,valuequery,key,value， 而softmaxsoftmaxsoftmax 的作用是使得输出权重的概率分布和为1

# 2.问：Transformer里自注意力机制的计算过程是怎样的？

答：

  1. 将输入单词列表(句子)转化成嵌入向量列表[word1,word2,word3.....wordn][word_1,word_2,word_3.....word_n][word1​,word2​,word3​.....wordn​]
  2. 根据嵌入向量得到 qqq ,kkk ,vvv三个向量以及对应的列表,[q1,q2...qn],[k1,k2...kn],[v1,v2...vn][q_1,q_2...q_n],[k_1,k_2...k_n],[v_1,v_2...v_n][q1​,q2​...qn​],[k1​,k2​...kn​],[v1​,v2​...vn​]；self-attention 中，q,k,vq,k,vq,k,v由 embedding 的结果经过不同的线性变换得到，维度都是 hhidden×lsrch_{hidden} \times l_{src}hhidden​×lsrc​ (前两步都是在自注意力计算之前)；
  3. 为每个向量计算一个score：score=q⋅kscore=q\cdot{k}score=q⋅k ；(q∗[k1,k2..kn]q*[k_1,k_2..k_n]q∗[k1​,k2​..kn​])
  4. 为了梯度的稳定，Transformer使用了score归一化，具体论证见下文,即除以 dk\sqrt{d_{k}}dk​​ ；
  5. 对score施以softmax激活函数；
  6. softmax点乘Value值 vvv ，得到加权的每个输入向量的评分 vvv ；
  7. 相加之后得到最终的输出结果 z:z=∑i=1nvz:z=\sum_{i=1}^{n}{v}z:z=∑i=1n​v 。

# 3.问：Multi-Head Attention是什么，有什么作用？

答：Multi-Head Attention相当于 hhh 个不同的self-attention的集成（ensemble），在这里我们以 举例说明。Multi-Head Attention的输出分成3步

  1. 将数据 XXX分别输入到8个self-attention中，得到8个加权后的特征矩阵 。
  2. 将8个 按列拼成一个大的特征矩阵；
  3. 特征矩阵经过一层全连接后得到输出 。

原论文中认为，将模型分为多个头，期望其形成多个相互独立子空间，可以让模型去关注不同方面的信息。但是也有很多论文不这么认为。也有人认为是计算复杂度的取舍（BERT中，multi-head 与直接使用768*768矩阵统一计算，有什么区别？ - 苏剑林的回答 - 知乎 https://www.zhihu.com/question/446385446/answer/1752279087），同时multi-head相对来说也比较冗余，mask掉一定比例的head对结果影响不大。

# 4.问：Transformer的中的Attention机制，其中Self-Attention和Encoder-DecoderAttention之间的关系？

答：

Encoder-Decoder Attention如图所示，编码器一般有两层，自注意力和前馈神经网络（不算正则化和残差连接），解码器则一般有三层：

  • 自注意力层
  • Encoder-Decoder Attention 层
  • 与位置无关的前馈网络层

在解码器中，Decoder block比Encoder中多了个Encoder-Decoder Attention。在Encoder-Decoder Attention中， QQQ来自上一层的自注意力层， 而其则从编码器的输出中获取KKK 矩阵和 VVV矩阵。其计算方式完全和encoder的过程相同。

因为在机器翻译这类问题里，解码过程是一个顺序操作的过程，也就是当解码第 个特征向量时，我们只能看到第 及其之前的解码结果，类似于LSTM根据前一个来生成后一个的过程。所以也把这种情况下的multi-head attention叫做masked multi-head attention。

# 5.问：都有哪些不同类型的注意力，其公式及计算过程和复杂度差异有什么区别。

这个问题也会被分为几个子问题

  1. 加性注意力(Additive Attention、乘性注意力(Dot Product Attention),缩放点积注意力(Scaled Dot-Product Attention)的公式与计算过程？
  2. Transformer计算attention的时候为何选择点乘而不是加法？
  3. 两者计算复杂度和效果上有什么区别？
  4. 为什么在进行softmaxsoftmaxsoftmax之前需要对attention进行scaled (attention计算时为什么要除dk\sqrt{d_k}dk​​ ) ？

答：这几个问题在理解上紧密相连，所以我们放在一起进行讨论。

首先我们来看一下三者的公式，

Additive Attention(加性注意力模型)

s(xi,q)=vTtanh(Wxi+Uq)(2)s(x_i,q)=v^{T}tanh(W{x_i}+Uq) \tag{2} s(xi​,q)=vTtanh(Wxi​+Uq)(2)

dot product attention （乘性注意力/点积模型）

S(xi,q)=xiTq(3)S(x_i,q)=x^{T}_{i}q \tag{3} S(xi​,q)=xiT​q(3)

Scaled Dot-Product Attention(缩放点积模型)：

S(xi,q)=xiTq/dk(4)S(x_i,q)=x^{T}_{i}q/\sqrt{d_k} \tag{4} S(xi​,q)=xiT​q/dk​​(4)

当然常见的还有 很多其他模型，如双线性模型 $ S(x_i,q)=x^{T}_{i}Wq $ 等：这里主要讨论上述三种

计算过程详解：

想象成一句中文与其对应的英文翻译，每个词为一个向量,两个序列也可以不等长，我们要计算两组序列彼此之间的attention权重。其过程大体可以被抽象为三步：

① 信息输入：将 语言输入模型，转换为对应的向量，q,k,vq,k,vq,k,v 或者 key,valuekey,valuekey,value 我们这里假设存在两组向量序列来表示一句话与其对应的翻译。

H:h1,h2,...hn\textbf{H}:h_1,h_2,...h_nH:h1​,h2​,...hn​和S:s1,s2,...,sn\textbf{S}:s_1,s_2,... ,s_nS:s1​,s2​,...,sn​，任意一个元素的维度为dkd_kdk​

② 对于任意的hi,sjh_i,s_jhi​,sj​,以某种attention函数计算两者之间的分数

③ 信息加权平均

这个过程对于Additive Attention而言，即为

1.计算任意向量hih_ihi​与向量s1,s2,...,sns_1,s_2,... ,s_ns1​,s2​,...,sn​的attention权重

2.然后拼接hi−s1h_i-s_1hi​−s1​通过一个特定的线性层计算出其分数，再拼接hi−s2h_i-s_2hi​−s2​重复通过线性层计算，...，直到所有的均计算出一个结果

3.再把所有结果通过softmaxsoftmaxsoftmax。

其计算过程公式如下

ct=∑i=1nαt,ihi;输出的上下文向量ytαt,i=align⁡(yt,xi);两个单词yt和xi的对齐情况=exp⁡(score⁡(st−1,hi))∑i′=1nexp⁡(score⁡(st−1,hi′));对自定义的对齐分数进行softmax(5)\begin{aligned} \mathbf{c}_{t}=&\sum_{i=1}^{n} \alpha_{t, i} \boldsymbol{h}_{i} \quad ; 输出的上下文向量 y_{t}&\\ \alpha_{t, i}=&\operatorname{align}\left(y_{t}, x_{i}\right) \quad; 两个单词y_{t}和x_{i}的对齐情况&\\ =&\frac{\exp \left(\operatorname{score}\left(s_{t-1}, \boldsymbol{h}_{i}\right)\right)}{\sum_{i^{\prime}=1}^{n} \exp \left(\operatorname{score}\left(\boldsymbol{s}_{t-1}, \boldsymbol{h}_{i^{\prime}}\right)\right)} \quad ; 对自定义的对齐分数进行 softmax& \end{aligned} \tag{5} ct​=αt,i​==​i=1∑n​αt,i​hi​;输出的上下文向量yt​align(yt​,xi​);两个单词yt​和xi​的对齐情况∑i′=1n​exp(score(st−1​,hi′​))exp(score(st−1​,hi​))​;对自定义的对齐分数进行softmax​​(5)

从这个角度讲，Additive Attention注意力的公式也可以看作

fscore(i,j)=vaTtanh⁡(Wa[hi;sj])=vTtanh⁡(W1si+W2hj])(6)f_{score(i,j)} = v_{a}^{T}\tanh\left(\textbf{W}_{a}\left[\textbf{h}_{i};\textbf{s}_{j}\right]\right)=\left.v^{T} \tanh \left(W_{1} s_{i}+W_{2} h_{j}\right]\right) \tag{6} fscore(i,j)​=vaT​tanh(Wa​[hi​;sj​])=vTtanh(W1​si​+W2​hj​])(6)

其中vav_ava​和Wa\textbf{W}_{a}Wa​是学习的注意参数。这里h\textbf{h}h是指编码器的隐藏状态，s\textbf{s}s是解码器的隐藏状态。

Dot Product Attention and Scaled Dot-Product Attention

同样地，我们也可以将fscore(i,j)f_{score(i,j)}fscore(i,j)​函数替换为乘性注意力，即

fscore(i,j)=F(si,hj)=siThj(7)f_{score(i,j)}=F\left(s_{i}, h_{j}\right)=s_{i}^{T} h_{j} \tag{7} fscore(i,j)​=F(si​,hj​)=siT​hj​(7)

而上述的计算过程不变，依次对任意的si,hjs_i,h_jsi​,hj​求出其乘性注意力分数,然后统一再softmaxsoftmaxsoftmax。细心的同学也应该已经发现了，这个过程对于序列H\textbf{H}H和S\textbf{S}S而言,就是矩阵相乘的过程。

Dot Product Attention 和 Additive Attention两者在复杂度上是相似的，Additive Attention增加了三个可学习的矩阵，所以相比另外两个效果会更好，同时也增加了更多的模型参数，计算效率会较低。而且Dot Product Attention因为是直接坐矩阵运算，在实际情况下一般会更快一些与高效一些，因为一般的深度学习框架底层都有对矩阵运算的一些优化，有很多的并行算法和硬件加速，比如5X5的矩阵相加，框架底层可能会是25个数字同时计算。手动计算则是for循环一个一个相加。速度就会差上很多。而Self Attention则如上文所讲，对于一句话引入了q,k,vq,k,vq,k,v三个矩阵，也保证了计算速度,所以综合来看是最好的。其计算过程如上述内容。

同时我们也可以看到，对于任意两个维度为dkd_kdk​的元素si,hjs_i,h_jsi​,hj​相乘，在dkd_kdk​较小时二者表现相似，但是dkd_kdk​较大时，Dot Product Attention表现不如Additive Attention，因为dkd_kdk​较大时点积之后的结果较大，方差也会变大，求softmax之后，梯度会很小,容易梯度消失，不利于计算，需要做一定的缩放。所以才会有了进一步的Scaled Dot-Product Attention

S(xi,q)=xiTq/dk(8)S(x_i,q)=x^{T}_{i}q/\sqrt{d_k} \tag{8} S(xi​,q)=xiT​q/dk​​(8)

除以dk\sqrt{d_k}dk​​的具体证明则如下(参考自paper原文)，

假设输入元素xix_ixi​转化而来的q,kq,kq,k每一维是均值为0、方差为1的独立随机变量，一共有dkd_kdk​维，那么它们的点积q⋅k=∑i=1dkqikiq·k=\sum_{i=1}^{d_k}q_ik_iq⋅k=∑i=1dk​​qi​ki​的均值为0、方差为dkd_kdk​

因为，如果两个变量XXX和YYY是独立随机变量，则其乘积的方差与期望为

var⁡(∑xiyi)=kvar⁡(x1y1)=kE[x12]E[y12]=kσ4(9)\operatorname{var}\left(\sum x_{i} y_{i}\right)=k \operatorname{var}\left(x_{1} y_{1}\right)=k E\left[x_{1}^{2}\right] E\left[y_{1}^{2}\right]=k \sigma^{4} \tag{9} var(∑xi​yi​)=kvar(x1​y1​)=kE[x12​]E[y12​]=kσ4(9)

E[∑xiyi]=kE[x1y1]=0(10)E\left[\sum x_{i} y_{i}\right]=k E\left[x_{1} y_{1}\right]=0 \tag{10} E[∑xi​yi​]=kE[x1​y1​]=0(10)

所以换算到上式中即方差为dkd_kdk​,我们要控制方差为1，所以就要除以dk\sqrt{d_k}dk​​，即下式

D(q⋅kdk)=dk(dk)2=1(11)D\left(\frac{q \cdot k}{\sqrt{d}_{k}}\right)=\frac{d_{k}}{\left(\sqrt{d}_{k}\right)^{2}}=1 \tag{11} D(d​k​q⋅k​)=(d​k​)2dk​​=1(11)

将方差控制为1，也就有效地控制了前面提到的梯度消失的问题。

# 6.问：self-attention代码实现？

答：


import torch.nn as nn

def clones(module, N):
    """
    Produce N identical layers.
    """
    return nn.ModuleList([copy.deepcopy(module) for _ in range(N)])


def attention(query, key, value, mask=None, dropout=None):
    """
    Compute 'Scaled Dot Product Attention'
    """
    d_k = query.size(-1)
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)
    p_attn = F.softmax(scores, dim=-1)
    if dropout is not None:
        p_attn = dropout(p_attn)
    return torch.matmul(p_attn, value), p_attn


class MultiHeadedAttention(nn.Module):
    """
    Implements 'Multi-Head Attention' proposed in the paper.
    """

    def __init__(self, h, d_model, dropout=0.1):
        """
        Take in model size and number of heads.
        """
        super(MultiHeadedAttention, self).__init__()
        assert d_model % h == 0
        # We assume d_v always equals d_k
        self.d_k = d_model // h
        self.h = h
        self.linears = clones(nn.Linear(d_model, d_model), 4)
        self.attn = None
        self.dropout = nn.Dropout(p=dropout)

    def forward(self, query, key, value, mask=None):
        if mask is not None:
            # Same mask applied to all h heads.
            mask = mask.unsqueeze(1)
        nbatches = query.size(0)

        # 1) Do all the linear projections in batch from d_model => h x d_k
        query, key, value = [l(x).view(nbatches, -1, self.h, self.d_k).transpose(1, 2)
                             for l, x in zip(self.linears, (query, key, value))]

        # 2) Apply attention on all the projected vectors in batch.
        x, self.attn = attention(query, key, value, mask=mask, dropout=self.dropout)

        # 3) "Concat" using a view and apply a final linear.
        x = x.transpose(1, 2).contiguous().view(nbatches, -1, self.h * self.d_k)
        return self.linears[-1](x)
